### 📖 Cxdec V2 (Hxv4) 加密技术:

Cxdec V2（业内更常称**Hxv4**）是Wamsoft在2020年底推出的新一代Kirikiri(krkrz)引擎加密方案，是对早期Cxdec V1的全面升级，核心采用**双重AES加密+全文件名哈希化**架构。

**核心加密流程**
<pre>
1.  第一层：Hxv4头部加密
    - 游戏主封包 `shturl.c` 的索引表（entry table）被 **AES-128** 单独加密
    - 解密密钥硬编码在游戏主程序EXE或魔改的 `shturl.c` 中，**每款游戏独立生成，无通用密钥**
    - 必须先逆向提取该密钥，才能解密出XP3封包的文件索引

2.  第二层：单文件内容加密
    - 解密索引表后，每个文件entry会生成对应的 `filter key`
    - 使用该key对文件内容进行二次AES-128加密
    - 脚本文件(.ks/.scn)会额外进行字节混淆处理

3.  最具破坏性的设计：全文件名哈希化
    - 封包中**不存储任何原始文件名**，仅存储文件名的 **SHA-1哈希值**
    - 引擎运行时，通过计算所需文件的哈希值来查找对应资源
    - 这是Cxdec V2与所有传统krkrz加密最本质的区别

4.  额外防护措施
    - 主程序普遍加 **VMProtect/Themida** 强壳，带反调试和反内存Dump
    - 引擎内置全文件完整性校验，修改任何文件都会导致游戏崩溃
    - 部分2024年后新作加入了在线资源校验机制
</pre>
---

**加密后的直观特征**
<pre>
1.  未解包时
    - 文件扩展名仍然是标准的 `.xp3`，外观和普通krkrz游戏完全一致
    - 用普通解包工具（GARbro、原版KrkrExtract）打开时：
      - 显示"无法识别的封包格式"
      - 或解包出0字节的空文件
      - 或直接崩溃报错

2.  仅解密头部后（无哈希表）
    解包后会看到满屏乱码哈希文件名，完全无法识别内容：
    ```
    shturl.cc/73U7QCBTTpzPYbUO4NgKYSSAqjfkl23
    shturl.cc/L03HRRNCeYMReOX9NN6U5Euj52SUaYLQIEv548C
    shturl.cc/btEzm1rkOo6w0VJLQEo5XFOpKvOGAZ5
    ...
    ```
    - 没有任何文件夹结构，所有文件都平铺在根目录
    - 无法区分脚本、CG、语音、UI资源
    - 即使能解密文件内容，也不知道哪个是主线剧情脚本

3.  解密文件内容后
    - 脚本文件打开全是乱码，无任何可读文本
    - 图片文件无法用看图软件打开，显示"文件损坏"
    - 音频文件无法播放，播放器提示格式不支持

4.  引擎运行时
    - 任何修改过的文件都会触发校验失败，游戏直接黑屏或弹出"文件损坏"提示
    - 无法通过替换同名文件的方式修改游戏内容
</pre>
